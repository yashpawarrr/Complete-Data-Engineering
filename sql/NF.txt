### BCNF ---> Boyce codd NF >>>>

Strict than 3-NF and aka 3.5 NF !

>>>> For any dependency LIKE ( A ----> B )
>>>> *A* should be a SUPER KEY !

(A super key is any set of one or more columns (attributes) that can uniquely identify a row (tuple) in a table)

Important point:
A super key may contain extra attributes
A minimal super key (no extra attributes) is called a candidate key

Super Key → Unique, may have extra columns
Candidate Key → Minimal unique set (no extra columns)


ex. 

roll_no   std_id   branch_id   branch_name

>> std_id *depends on* roll_no  &&  branch_name on branch_id 

>>>> one stu. can opt two branch >>> its not super key !!!
